Homework 1: Tasks for iwc3

Your personal task is to develop a system for shops and their products.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Product. With the following private fields:

  1. String packaging
  2. String supplier
  3. String wares
  4. int price
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Shop. With the following private fields:

  1. int id
  2. String city
  3. String founder
  4. List<Product> products
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Shop> shops in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one shop with two products. [4 marks]

Create controller co2123.hw1.controller.ShopController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/shops" should show view shops/list. [2 marks]
    • Add the list of all shops from co2123.hw1.Hw1Application to the model as attribute shops. [2 marks]
    • Create the corresponding JSP to list the city and founder of each shop followed by a link to URL "/products?shop={id of shop}", e.g., "/products?shop=3". [4 marks]
  2. Requests for URL "/newShop" should show view shops/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="shop" and action="/addShop" and inputs for all fields of class Shop except for products. [5 marks]
  3. POST requests for URL "/addShop" should add a new shop to the list of all shops in co2123.hw1.Hw1Application and redirect to /shops. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.ProductController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/products" should show view products/list. [2 marks]
    • Based on request parameter shop determine the corresponding shop from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all products of that shop to the model as attribute products. [2 marks]
    • Create the corresponding JSP for the view to list the packaging, supplier, wares, and price of each product. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newProduct?shop={id of shop whose products are listed}", e.g., "/newProduct?shop=3". [4 marks]
  2. Requests for URL "/newProduct" should show view products/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="product" and action="/addProduct" and inputs for all fields of class Product. [6 marks]
    • Also add the request parameter shop with the ID of the current shop. [4 marks]
  3. POST requests for URL "/addProduct" should add a new product to the list of products of the shop identified by request parameter shop and then redirect to /shops. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Shop and make it available in controller ShopController. [4 marks]

When adding a new shop validate that

  1. The value of field id is unique within the list Hw1Application.shops. [3 marks]
  2. The value of field city is not empty. [2 marks]
  3. The value of field founder is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Product and make it available in controller ProductController. [4 marks]

When adding a new product validate that

  1. The value of field packaging is not empty. [2 marks]
  2. The value of field supplier is not empty. [2 marks]
  3. The value of field wares is either "Water", "Juice", or "Smoothies". [3 marks]
  4. The value of field price is between (including) 1 and 70. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
